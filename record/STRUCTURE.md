# 时间轴文件结构规范

## 目录结构

```
record/
├── [年份]/
│   ├── [月份]/
│   │   ├── [地点]/
│   │   │   ├── Day1/
│   │   │   │   ├── 图片1.jpg
│   │   │   │   └── ...
│   │   │   ├── Day2/
│   │   │   │   └── ...
│   │   │   └── [其他文件夹或图片]
│   │   └── ...
│   └── ...
└── 时间轴.png
```

## 现有结构示例

record/

## 添加新记录的规范

### 命名规则
- **年份文件夹**: 4位数字，如 `2026`
- **月份文件夹**: 数字 + "月"，如 `4月`、`12月`
- **地点文件夹**: 自定义名称，如 `湖州`、`杭州`、`日本`
- **天数文件夹**: `Day1`、`Day2`... 或自定义

### 示例：添加"26年4月去湖州"

在 `record/2026/4月/` 下新建 `湖州` 文件夹：

```
record/
└── 2026/
    └── 4月/
        └── 湖州/
            ├── Day1/
            │   └── [湖州Day1的图片]
            └── Day2/
                └── [湖州Day2的图片]
```

### 图片格式
- 支持格式: jpg、jpeg、png、gif、webp
- 建议按日期/天数分组存放
- 封面图通常使用 Day1 文件夹中的第一张图片

## 数据读取逻辑

程序按以下规则自动扫描目录生成时间轴：

1. 扫描 `record/` 下所有年份文件夹
2. 每个年份下扫描月份文件夹
3. 每个月份下扫描地点文件夹
4. 每个地点下的图片自动收集
5. 封面图取该地点 Day1 文件夹的第一张图片
